# Purpose

VendorNote associates Vendor records with reusable Note entities maintained within the Shared Kernel.

This table allows users to attach unlimited business notes to a Vendor without duplicating note storage.

Typical notes include:

- Procurement discussions
- Vendor evaluation comments
- Payment follow-up
- Quality observations
- Compliance remarks
- Internal purchasing comments
- Contract negotiation history
- Blacklisting reasons
- Audit observations

The actual note content is stored inside the Shared Kernel **Note** table while VendorNote simply establishes the business relationship.

# Business Rules

- Unlimited notes may be attached to a Vendor.
- A Vendor cannot reference the same Note more than once.
- Notes remain reusable within the Shared Kernel.
- Deleting VendorNote does not delete the Note.
- Vendor ownership belongs to Vendor Domain.
- Soft Delete only.
- Audit fields mandatory.
- RowVersion mandatory.

# Workflow

1. User opens Vendor.
2. Opens **Notes** tab.
3. Creates or links an existing Shared Note.
4. System inserts VendorNote bridge record.
5. Note immediately becomes visible under Vendor history.

# Developer Notes

- Implements the Shared Kernel Bridge Pattern.
- No business text is stored in this table.
- Note content remains centralized inside the Shared Kernel.
- Supports unlimited reusable notes.
- Prevents duplication of note records.
- Fully consistent with VendorAttachment, VendorActivity and VendorTimeline.
